# From Chaos to Control: My Linux Journey with CachyOS

_Gaming, coding, editing — all finally under my control._

---

## The Day I Ditched Windows

Like most devs, I started on Windows.

It was familiar. Predictable. But over time, it became a drag — background updates, weird errors, bloated UI, and that constant feeling of being watched.

I wanted something that felt more like **a tool**, not a service.

So I did it. I jumped into the unknown — I installed Linux.

---

## The Rough Beginning: Confusion, Crashes & Constant Switching

I’ll keep it real — my first month on Linux was **chaotic**.

I tried a random distro. It didn’t even survive the first boot. Then I fell into a cycle:

- Install
- Break
- Reinstall
- Repeat

**Desktop Environments I tried in a single week:**

- `GNOME` – Looked modern, but stiff
- `XFCE` – Lightweight, but dry
- `Cinnamon` – Smooth-ish, but not “me”

I copied configs off Reddit and GitHub without even understanding them. My GRUB broke **twice**. My brain? Constantly fried.

> “It felt like trying to assemble a spaceship with IKEA instructions — written in Klingon.”

## Discovering CachyOS: The Turning Point

Everything changed when I found **CachyOS**.

This distro hit different. It didn’t just boot — it **flew**. It came pre-optimized and ready for modern workflows.

### Why CachyOS worked:

- ✅ Performance-focused Arch base
- ✅ KDE Plasma done right (smooth + fast)
- ✅ BTRFS snapshots with Timeshift preconfigured
- ✅ Flatpak support, ZRAM, gaming tweaks, all built-in
- ✅ Modern kernel (`cachy`) with real-world optimizations

> It wasn’t just an OS — it was a foundation to build my entire digital workflow on.

---

## Building the Dream Setup

Once I had stability, I started crafting my system around the **4 pillars** that matter to me:

---

### 🎮 1. Gaming That Doesn’t Suck

- **Heroic Games Launcher** → for Epic & GOG
- **Proton + Wine + Protontricks** → for compatibility
- **Gamemode** → auto performance boost
- **MangoHUD** → clean in-game FPS overlay
- **VKD3D & DXVK** → smooth DirectX translations

> Games that lagged on Windows were running smoother on Linux. Wild.

---

### 💻 2. Coding on Steroids

- **Frontend**: React 19 + Vite + Tailwind CSS v4
- **Backend**: Express + MongoDB
- **Dev Tools**: VS Code + Neovim, ESLint, shadcn/ui
- **Terminal**: ZSH + Starship + Alacritty
- **Package Managers**: pacman + yay + pnpm

> My dev environment is fast, clean, and modular. And I control every part of it.

---

### 🎬 3. Editing & Content Creation

- **Kdenlive** – non-linear editing, GPU-accelerated
- **OBS Studio** – recording & streaming
- **GIMP** – image editing
- **Handbrake** – fast encoding & compression

> No crashes, no locked codecs, no license nags — just creative freedom.

---

### 📈 4. Productivity, but Beautiful

- **KDE Plasma** + Latte Dock → aesthetic & functional
- **Custom keybinds** for everything
- **Window rules** for organized multitasking
- **KRunner**, clipboard manager, night light, session restore

> It looks amazing. But more importantly — it _works_ the way I think.

---

## Lessons from the Journey

This wasn’t just an OS change. It was a **mindset shift**.

Linux taught me to:

- Be **curious**, not scared
- Break things, then **learn how to fix them**
- Stop searching for “the perfect distro” and start building my own system

> “Linux isn't hard — it just doesn’t hold your hand. But once you figure it out, you won’t want anything else.”
